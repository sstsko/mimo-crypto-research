MiMo Crypto Research is an event-driven, multi-agent system for automated cryptocurrency token analysis. The core architecture uses an asynchronous message bus to decouple agents from one another: a central scheduler dispatches scan events, each agent publishes its findings as structured messages, and a shared SQLite database persists state across restarts. This design allows individual agents to fail and restart independently without disrupting the overall pipeline, and new agents can be added by simply subscribing to the relevant message topics.

The system comprises five specialised agents working in concert. The Discoverer identifies new and trending tokens across multiple chains by querying DEX aggregators and processing the seed list. The ContractChecker verifies on-chain contract source code, detects proxy patterns, and performs honeypot simulation. The RiskAnalyst consumes the outputs of the first two agents and produces a composite risk verdict using liquidity depth, holder distribution, and contract heuristics. The Reporter synthesises all gathered data into structured scan reports and stores them for downstream consumption. Finally, the AlertWatcher monitors open positions and watchlist entries, emitting real-time notifications when price thresholds or risk-level changes are detected.

In production the system is powered by MiMo V2.5 (Xiaomi's reasoning model) for natural-language tasks such as interpreting contract decompiler output and generating human-readable risk summaries. MiMo V2.5 is invoked via the Xiaomi API and its token usage is tracked per-agent for cost monitoring. The entire stack runs on a single Linux host, scheduled through systemd timers, and requires no external message broker — the event bus and task queue are implemented in-process using Python asyncio primitives, keeping operational overhead minimal while still providing the resilience and modularity of a distributed architecture.
